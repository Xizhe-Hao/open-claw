### 5️⃣ 安全隐患检查 ⚠️ 需要改进

#### 检查项目

| 检查内容 | 文件 | 状态 | 风险等级 |
|---------|------|------|---------|
| **API密钥暴露** | login_api.js | ⚠️ 风险 | 🔴 高 |
| **密码硬编码** | login_api.js | ⚠️ 风险 | 🔴 高 |
| **CORS配置** | login_api.js | ⚠️ 风险 | 🟡 中 |
| **环境变量** | login_api.js | ⚠️ 风险 | 🔴 高 |
| **用户数据存储** | login_api.js | ⚠️ 风险 | 🟡 中 |
| **Token安全** | login_api.js | ✅ 安全 | 🟢 低 |

#### 详细问题分析

**🔴 高危问题#1: JWT密钥硬编码**
```javascript
// ❌ 不安全（行号23）
const JWT_SECRET = process.env.JWT_SECRET || 'your-secret-key-change-in-production';
```
**风险**: 如果JWT_SECRET环境变量未设置，默认值会暴露  
**影响**: 攻击者可用此密钥伪造任何用户的Token  
**建议**:
```javascript
// ✅ 安全做法
const JWT_SECRET = process.env.JWT_SECRET;
if (!JWT_SECRET) {
  throw new Error('JWT_SECRET环境变量未设置，无法启动服务');
  process.exit(1);
}
```

**🔴 高危问题#2: 密码哈希值硬编码在代码中**
```javascript
// ❌ 不安全（行号37-46）
const users = new Map([
    ['user1@example.com', {
        // ... 
        passwordHash: '$2b$10$qJ7q8H8K9L0P2Q3R4S5T6...',
    }]
]);
```
**风险**: 测试用户密码被固定在代码中，易被暴露  
**影响**: 任何获得源码的人都能登录测试账户  
**建议**:
```javascript
// ✅ 改进做法
// 1. 改用真实数据库（MongoDB/PostgreSQL）
// 2. 测试环境密钥通过环境变量加载
// 3. 生产环境与开发环境分离
```

**🔴 高危问题#3: 环境变量未在.env.example中定义**
```javascript
// ❌ 不安全
require('dotenv').config();
const ALLOWED_ORIGINS = process.env.ALLOWED_ORIGINS || [...];
```
**风险**: 新开发者不知道需要配置哪些环境变量  
**建议**: 创建`.env.example`文件：
```
JWT_SECRET=your_jwt_secret_key_here
PORT=3001
ALLOWED_ORIGINS=http://localhost:3000,https://yourdomain.com
NODE_ENV=development
```

**🟡 中危问题#4: CORS配置过于宽松**
```javascript
// ⚠️ 可改进（行号15-17）
app.use(cors({
    origin: process.env.ALLOWED_ORIGINS || ['http://localhost:3000', 'https://yoursite.com'],
    credentials: true
}));
```
**风险**: 如果ALLOWED_ORIGINS环境变量未设置，会允许默认源  
**建议**: 
```javascript
// ✅ 改进
const allowedOrigins = (process.env.ALLOWED_ORIGINS || '').split(',').filter(Boolean);
if (allowedOrigins.length === 0 && process.env.NODE_ENV === 'production') {
  throw new Error('ALLOWED_ORIGINS未配置');
}
app.use(cors({ origin: allowedOrigins, credentials: true }));
```

**🟡 中危问题#5: 内存存储不持久化**
```javascript
// ⚠️ 问题（行号37）
const users = new Map([...]);  // 内存Map，重启即丢失
```
**风险**: 用户数据无法持久化，服务重启后数据丢失  
**影响**: 生产环境无法使用  
**建议**: 集成真实数据库
```javascript
// ✅ 改进（伪代码）
const User = require('./models/User'); // MongoDB model
const user = await User.findOne({ email });
```

**✅ 合格项: JWT Token安全**
- Token采用HS256签名
- 有效期7天合理
- 过期自动失效
- 不存储敏感信息
